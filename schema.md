# News Atom Lite Schema Reference

---

## Event

An event is a discrete real-world happening. It exists independently of any single article. The same event can be referenced by atoms from many different articles over time.

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Always `"event"` |
| `event_label` | string | Yes | `"YYYY-MM-DD [AGENT] [ACTION] [OBJECT] [@LOCATION]"` — max 12 words |
| `date` | string | Yes | ISO 8601. `"undated"` if not determinable. Partial dates allowed: `"2025-01"`, `"1940s"` |
| `agent` | string | Yes | Causal agent. `"unknown"` if unstated. Never the grammatical subject of a passive construction |
| `participants` | array | Yes | Entities named as involved. Minimum one entry |
| `action` | string | Yes | Root verb. Extract from passive constructions: `"was renamed"` → `"renamed"` |
| `object` | string | Yes | What the action is directed at. Concise |
| `location` | string | No | Where the event occurred. Omit if not determinable |
| `related_event` | string | No | The `event_label` of another event this explicitly connects to |
| `relation_phrase` | string | No | Exact connecting phrase from the text, verbatim. Only when `related_event` is set |

### event_label format

```
YYYY-MM-DD [AGENT] [ACTION] [OBJECT] [@LOCATION]
```

Examples:
```
2025-04-01 Dhami renamed 17 places @Uttarakhand
2024-11-21 Adani Group cancelled $600-million bond offering
undated Hindutva groups demanded Aurangzeb tomb removal @Maharashtra
```

When the agent is unknown, use the primary participant instead.

### agent rules

The agent is the **causal** agent, not the grammatical subject.

| Sentence | Correct agent |
|----------|---------------|
| `"The wall collapsed during heavy rain"` | `"heavy rain"` |
| `"The bill was passed by Parliament"` | `"Parliament"` |
| `"Police registered an FIR"` | `"police"` |
| `"Seven persons were killed"` | `"unknown"` |

## Atom

An atom is a sentence-level record of how journalism documented an event. **One sentence = one atom, always.**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `type` | string | Yes | Always `"atom"` |
| `event_label` | string or array | Yes | Matching `event_label` from events. String if one event, array if multiple |
| `information_type` | string | Yes | `"observed_fact"` or `"sensemaking"` |
| `direct_quote` | boolean | Yes | `true` if and only if typographic double quote marks present in the sentence |
| `source` | string | No | Exact attribution phrase verbatim. Only when `direct_quote` is `true` |
| `subject` | string | Yes | Grammatical subject of the sentence |
| `predicate` | string | Yes | Main verb or verb phrase. 1–5 words |
| `object` | string or array | Yes | What the action is directed at. Concise |
| `date` | string | No | Resolved ISO 8601. Omit if no temporal reference in sentence |
| `location` | string | No | Where the action occurs. Stated in sentence. Omit otherwise |
| `original_text` | string | Yes | Exact verbatim sentence. Never modified, truncated, or paraphrased |
| `entities` | array | Yes | Array of `{"name": string, "type": string}` |
| `language` | string | Yes | Language code e.g. `"en"` |

### information_type

**`observed_fact`** — the journalist is a witness or conduit of a witness. The sentence reports an action, statement, or event that occurred.

**`sensemaking`** — the journalist is an explainer. Definitions, background context, analysis, uncertainty.

Uncertainty sentences are always `sensemaking`:
- `"It was not immediately clear whether..."` → `sensemaking`
- `"It remains to be seen..."` → `sensemaking`

### direct_quote

`true` if and only if typographic double quote marks (`"..."`) appear in the sentence. Deterministic.

Indirect attribution is **not** a direct quote:
- `"the newspaper quoted X as saying"` → `direct_quote: false`
- `"X told reporters that Y happened"` → `direct_quote: false`

### source

Only when `direct_quote: true`. The attribution phrase exactly as it appears.  
Omit if `direct_quote` is `false`.  
Omit if `direct_quote` is `true` but no attribution phrase exists in the sentence.

### entities

Array of `{"name": string, "type": string}`.

| Type | Examples |
|------|---------|
| `person` | `"Gautam Adani"`, `"Indira Jaising"` |
| `organization` | `"Adani Group"`, `"Supreme Court"`, `"BSE"` |
| `location` | `"Maharashtra"`, `"Haridwar district"` |
| `event` | `"Waqf Amendment Bill"`, `"Pahalgam attack"` |
| `concept` | `"Foreign Corrupt Practices Act"`, `"Prevention of Money Laundering Act"` |
| `quantity` | `"$265 million"`, `"20.61%"` |

Extract institutional nouns when referring to specific bodies: `"police"`, `"court"`, `"hospital"`, `"government"`.

| Field | Description |
|-------|-------------|

## Key rules

1. **One sentence = one atom.** Terminal punctuation is the only boundary judge.
2. **Exception:** A direct quote block with a single attribution phrase is one atom regardless of internal punctuation.
3. **`event_label` must exactly match** a label in the events output.
4. **`agent` ≠ `subject`.** Agent is causal. Subject is grammatical. They differ in passive constructions — and that's correct.
5. **`direct_quote` is structural.** Quote marks present = `true`. No quote marks = `false`. No exceptions.
6. **Historical events belong in the events file** with their historical date.
7. **Preserve the journalist's exact verb.** `"acquires"` and `"buys"` are both valid for the same underlying event. Normalisation is downstream.
8. **`original_text` is sacred.** Never modify, paraphrase, or truncate it.
